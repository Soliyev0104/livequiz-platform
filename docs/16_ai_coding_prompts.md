# AI Coding Prompts

Use these prompts one by one. Do not ask an AI to build the whole system in one shot.

## Prompt 1 — Bootstrap repo

Create the `livequiz` monorepo structure exactly as specified in `docs/03_folder_structure.md`. Add Dockerfiles, pyproject files, package files, .gitignore, Makefile, and placeholder health endpoints. Do not implement business logic yet. Ensure `docker compose config` passes.

## Prompt 2 — Database models and migrations

Using `docs/04_domain_model_dbml.md` and `docs/05_database_schema_and_migrations.md`, implement SQLAlchemy models and Alembic migrations. Use BIGINT IDs generated by our Snowflake component, not autoincrement. Add seed scripts for demo users and quiz sets. Include tests that migrations apply from empty DB.

## Prompt 3 — Auth and roles

Implement register/login/refresh/logout/me endpoints with password hashing, JWT tokens, roles, and dependency helpers. Add role guards for host/moderator/admin. Include unit and integration tests.

## Prompt 4 — Quiz builder

Implement quiz-set CRUD, question CRUD, answer options, publish validation, tags, and search. Keep routers thin and business logic in services. Add OpenAPI schemas and tests.

## Prompt 5 — Rooms and lobby

Implement room creation, room code generation, join flow for guest and registered players, Redis room snapshot, capacity check, nickname uniqueness, and REST room snapshot endpoint.

## Prompt 6 — WebSocket live room

Implement `/ws/rooms/{room_code}` using `docs/07_websocket_protocol.md`. Add connection manager, Redis pub/sub bridge, heartbeat, reconnect snapshot, and message validation. Include tests for message parsing and a manual two-tab test guide.

## Prompt 7 — Match and scoring

Implement start match, question start/close, answer submission, scoring by correctness and response speed, Redis sorted-set leaderboard, final_scores persistence, and idempotency by request ID. Insert outbox events in the same transaction.

## Prompt 8 — Outbox + Redpanda + ClickHouse

Implement outbox publisher and stream worker from `docs/08_streaming_pipeline_and_bpmn.md`. Create Redpanda topics, ClickHouse migrations, event envelope validation, dedupe, and analytics endpoints.

## Prompt 9 — Moderation

Implement report creation, simple rule-based text flagging, moderator queue, moderation decisions, audit logs, and related events.

## Prompt 10 — Observability

Instrument API and workers with OpenTelemetry, Prometheus metrics, JSON logs, and trace IDs. Add Grafana/Prometheus/Loki/Tempo configs and dashboard JSON placeholders.

## Prompt 11 — Frontend

Build the frontend from `docs/13_frontend_ui_ux.md`: landing page, auth, dashboard, quiz builder, host room, player room, analytics, moderation. Use TypeScript and a clean component system. Connect REST and WebSocket clients.

## Prompt 12 — Final hardening

Add error handling, rate limits, CORS, security headers in Nginx, tests, load-test script, deployment docs, report screenshots checklist, and CHANGELOG. Make sure every project requirement R1–R13 is visibly satisfied.
